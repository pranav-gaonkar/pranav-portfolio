Union File System (FUSE)

Low-level systems project implementing a minimal Union File System using FUSE 3, demonstrating layered filesystem architecture similar to container storage systems like Docker.

  • Layered filesystem: upper (writable) + lower (read-only) with Copy-on-Write semantics
  • Whiteout files (.wh.<name>) for deletion handling in layered namespace
  • Core POSIX ops: getattr, readdir, read, write, create, unlink, mkdir, rmdir
  • Built on FUSE 3 for user-space filesystem integration on Linux/WSL2
