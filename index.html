Union File System (FUSE-based Layered Filesystem)

A low-level systems programming project implementing a minimal Union File System using FUSE 3, demonstrating layered file system architecture similar to container storage systems.

  • Implemented a layered filesystem with upper (writable) and lower (read-only) directories.
  • Designed and implemented Copy-on-Write semantics for modifying lower-layer files.
  • Implemented whiteout files (.wh.<name>) to handle deletions in layered namespace.
  • Supported core POSIX file operations including getattr, readdir, read, write, create, unlink, mkdir, and rmdir.
  • Built using FUSE 3 for user-space filesystem integration on Linux/WSL2.
  • Demonstrated real-world concepts used in container systems like Docker image layering.
